# LAUNCH-003: Release Coordination Plan

## Версия

1.0.0

## Статус

Активен

## Режим

Release Coordination.

## Правила

- **Hermes не вносит изменения в WordPress самостоятельно.**
- Все правки в WordPress/Elementor/Yoast выполняет **Operator** или **владелец** по инструкции Hermes.
- Hermes готовит контент, инструкции, проводит аудит и проверку результата.
- Цикл работы: **Подготовка → Внесение → Проверка → Закрытие**.

## Что разрешено Hermes

- Подготовка контента для страниц.
- Подготовка пошаговых инструкций.
- Технический аудит через curl, браузер, WP API.
- Проверка результата после внесения изменений.
- Ведение Launch Checklist.
- Формирование отчётов.
- Открытие задач на исправление при обнаружении проблем.

## Что запрещено Hermes

- Публикация/обновление страниц.
- Изменение меню.
- Изменение SEO-настроек.
- Изменение структуры сайта, URL, редиректов.
- Работа в админке WordPress.
- Работа с FTP/SSH для изменения файлов сайта.

## Процесс работы

### 1. Подготовка

- Hermes готовит контент и инструкции.
- Контент сохраняется в `Site_Launch/Page_Content_*.md`.
- Инструкции сохраняются в `Site_Launch/WordPress_Implementation_Guide.md`.
- Для каждого блока работ формируется отдельный GitHub Issue.

### 2. Внесение

- Operator или владелец выполняет инструкцию в WordPress.
- После завершения отмечает Issue как выполненный или сообщает Hermes.

### 3. Проверка

- Hermes проверяет результат через curl, браузер, WP API.
- Сравнивает фактический результат с ожидаемым.
- Фиксирует результаты в `Site_Launch/Verification_Report_*.md`.

### 4. Закрытие

- Если всё корректно — задача закрывается.
- Если обнаружены проблемы — открывается новая задача с описанием и повторяется цикл.
