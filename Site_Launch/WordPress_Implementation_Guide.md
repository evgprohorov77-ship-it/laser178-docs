# Инструкция для Operator по внесению контента на сайт laser178.ru

## Роль

Эта инструкция предназначена для Operator или владельца сайта. Hermes не вносит изменения в WordPress самостоятельно.

## Цель

Быстро и без ошибок внести подготовленный контент в страницы WordPress + Elementor.

## Источники контента

- `Page_Content_Contacts.md`
- `Page_Content_About.md`
- `Page_Content_Services.md`
- `Page_Content_Works.md`
- `SEO_Launch_Settings.md` — SEO-настройки и 301-редиректы

## Что понадобится

- Доступ в админку WordPress (admin).
- Плагин Elementor (уже установлен).
- Плагин Yoast SEO (уже установлен).
- Плагин для добавления кода в head/футер (опционально): Header Footer Code Manager, Insert Headers and Footers, или доступ к `functions.php`.

## После каждого шага

1. Сохранить/обновить страницу в WordPress.
2. Очистить кэш (если установлен плагин кэширования).
3. Проверить результат на сайте в режиме инкогнито.

## Шаг 1. Исправить меню (битая ссылка /bortovoy-zhurnal/)

1. В админке перейти **Внешний вид → Меню**.
2. Найти пункт с URL `/bortovoy-zhurnal/` (или ссылкой на страницу «Бортовой журнал»).
3. Заменить на правильный URL `/bortovoj-zhurnal/` или `/blog/`.
4. Сохранить меню.
5. **Критерий:** нажатие на пункт меню больше не ведёт на 404.

## Шаг 2. Включить страницы в XML sitemap (Yoast SEO)

1. В админке перейти **Yoast SEO → Настройки**.
2. Перейти в раздел **Карта сайта**.
3. В разделе **Типы контента** убедиться, что **Страницы** включены в sitemap.
4. Если отключены — включить.
5. Очистить кэш.
6. **Критерий:** открыть `https://laser178.ru/page-sitemap.xml` — должны появиться все страницы, а не только главная.

## Шаг 3. Страница «Контакты»

1. **Страницы → Контакты → Изменить (Elementor)**.
2. Удалить старый пустой/неполный контент.
3. Добавить текстовый блок с адресом, режимом работы, телефоном, мессенджерами из `Page_Content_Contacts.md`.
4. Вставить HTML-виджет с картой Яндекса (код в `Page_Content_Contacts.md`).
5. В Yoast SEO обновить Title и Meta description согласно `Page_Content_Contacts.md`.
6. Добавить JSON-LD микроразметку LocalBusiness в head страницы (код в `Page_Content_Contacts.md`).
7. **Критерий:** страница содержит адрес, карту, телефон, режим работы, JSON-LD в head.

## Шаг 4. Страница «О нас»

1. **Страницы → О нас → Изменить (Elementor)**.
2. Заменить пустой контент на текст из `Page_Content_About.md`.
3. В Yoast SEO обновить Title и Meta description.
4. H1: «О компании Лазер Антикор».
5. **Критерий:** страница содержит полный текст, уникальный H1 и мета-теги.

## Шаг 5. Страница «Услуги»

1. **Страницы → Услуги → Изменить (Elementor)**.
2. Заменить текущий минимальный контент на текст и таблицы из `Page_Content_Services.md`.
3. В Yoast SEO обновить Title и Meta description.
4. H1: «Услуги автосервиса Лазер Антикор».
5. **Критерий:** страница содержит описание услуг и таблицы с ценами, H1 и мета-теги корректны.

## Шаг 6. Страница «Наши работы»

1. **Страницы → Наши работы → Изменить (Elementor)**.
2. Заменить пустой контент на текст из `Page_Content_Works.md`.
3. Если есть фото — добавить карточки через виджеты «Заголовок + Текст + Изображение».
4. Добавить alt-теги к фото.
5. В Yoast SEO обновить Title и Meta description.
6. H1: «Примеры работ Лазер Антикор».
7. **Критерий:** страница содержит описание подхода и хотя бы шаблон карточки, H1 и мета-теги корректны.

## Шаг 7. Убрать дубли страниц через 301 редиректы

В WordPress существуют 4 пары дублей, обе версии доступны и возвращают 200. Рекомендуется вариант A (301 редиректы + удаление дублей).

### Вариант A — редиректы + удаление дублей (рекомендуется)

1. Установить плагин **Redirection** (или внести в `.htaccess` вручную).
2. Добавить редиректы:

| Source URL | Target URL |
|------------|------------|
| `/bortovoj-zhurnal/` | `/blog/` |
| `/galereja-do-posle/` | `/before-after/` |
| `/follow-model/` | `/sledit-za-modelju/` |
| `/proverit-ugrozu/` | `/risk-test/` |

3. Проверить, что редиректы работают (200 → 301 → 200).
4. Удалить дублирующиеся страницы: `bortovoj-zhurnal`, `galereja-do-posle`, `follow-model`, `proverit-ugrozu`.

### Вариант B — .htaccess

```apache
Redirect 301 /bortovoj-zhurnal/ /blog/
Redirect 301 /galereja-do-posle/ /before-after/
Redirect 301 /follow-model/ /sledit-za-modelju/
Redirect 301 /proverit-ugrozu/ /risk-test/
```

**Критерий:** открытие дублирующих URL даёт 301 и перенаправляет на основной URL.

## Шаг 8. Очистить кэш

После всех изменений:
1. Очистить кэш в плагине кэширования (WP Rocket, LiteSpeed, W3 Total Cache и т.п.).
2. Очистить кэш CDN, если используется.
3. Проверить страницы в режиме инкогнито.

## Шаг 9. Финальная проверка (перед сообщением Hermes)

Проверить каждую страницу в браузере:

- [ ] Главная открывается, title и H1 корректные.
- [ ] `/contacts/` — адрес, карта, телефон, режим работы.
- [ ] `/about/` — полный контент.
- [ ] `/services/` — услуги и цены.
- [ ] `/works/` — контент страницы.
- [ ] `/booking/` — форма работает, заявка доходит.
- [ ] `/blog/` или `/bortovoj-zhurnal/` — блог открывается из меню без 404.
- [ ] `/page-sitemap.xml` и `/post-sitemap.xml` содержат URL.
- [ ] Дубли больше не доступны (дают 301 или 404).

После завершения сообщить Hermes для проверки.

## Что делать, если что-то не получается

1. Не вносить изменения «на глазок».
2. Сделать скриншот проблемного места.
3. Создать Issue в репозитории `laser178-ai-framework` с описанием.
4. Hermes подготовит уточнённую инструкцию.
