## Security Review

### Модель угроз

Advisor — это внешний LLM, который получает текстовый контекст и возвращает рекомендацию. Основные угрозы:

1. **Утечка секретов** — API-ключи, пароли, токены могут попасть в третий сервис.
2. **Утечка PII** — email, телефоны клиентов могут быть логированы или отправлены в OpenAI.
3. **Переход от советов к действиям** — внешний LLM может предложить выполнить опасную команду, и кто-то примет это за инструкцию.
4. **Отсутствие отказоустойчивости** — при падении API важные HIGH-risk задачи могут пройти без одобрения владельца.
5. **Подделка ответа** — невалидный JSON или манипуляция схемой может привести к ложному одобрению.

### Реализованные контрмеры

| Угроза | Контрмера | Статус |
|--------|-----------|--------|
| Утечка секретов | API-ключ только из `OPENAI_API_KEY`, нет хранения в коде/YAML/.env | ✅ |
| Утечка секретов | Sanitization удаляет API-ключи, пароли, токены, cookie, env-файлы | ✅ |
| Утечка PII | Sanitization удаляет email и телефоны | ✅ |
| Утечка PII | В логи пишется только task_id, review_type, model, verdict, confidence, usage | ✅ |
| Действия вместо советов | Advisor не имеет доступа к инструментам, серверам, файлам | ✅ |
| Действия вместо советов | В промптах явно указано: «You do not have access to tools» | ✅ |
| Действия вместо советов | Verdict не равен «production action approved» | ✅ |
| Отказоустойчивость | При любой ошибке API возвращается `advisor_unavailable` | ✅ |
| Отказоустойчивость | Для HIGH-risk задачи fallback требует `requires_owner_decision: true` | ✅ |
| Подделка ответа | Все ответы валидируются по JSON Schema | ✅ |
| Подделка ответа | Invalid JSON → `advisor_unavailable` | ✅ |
| Контроль затрат | `enabled: false` по умолчанию | ✅ |
| Контроль затрат | Дневной лимит запросов | ✅ |
| Контроль затрат | Максимальный размер контекста и ответа | ✅ |
| Контроль затрат | Логирование usage (tokens, model) | ✅ |

### Рекомендации по безопасности для Owner

1. **Добавляйте OPENAI_API_KEY только в окружение сервера**, не в чат и не в файл репозитория.
2. **Перед включением `enabled: true` в production** прогоните dry run с реальным ключом в изолированном окружении.
3. **Настройте мониторинг** логов Advisor на наличие `advisor_authentication_error` или `advisor_rate_limited`.
4. **Регулярно проверяйте**, что в `sanitized_context` не утекают новые типы секретов (например, OAuth tokens, JWT).
5. **Не полагайтесь на Advisor как на единственный барьер** — он консультирует, но окончательное решение всегда остаётся за Hermes Policy Layer и Owner Approval.

### Результат Security Review

Модуль соответствует требованиям LAOS TASK. Включать в production можно только после ручного тестирования с реальным `OPENAI_API_KEY` и подтверждения Owner.
